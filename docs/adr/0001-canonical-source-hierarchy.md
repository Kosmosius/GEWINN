# ADR 0001: Canonical source hierarchy

## Status

Accepted during repository bootstrap.

## Decision

Use the ordered hierarchy in `docs/CANONICAL_SOURCES.md`: migration manifest, migration changelog, canonical data, encoded designer decisions, v5 procedures, v2.5 voice/identity/attribution/discrete wounds, then historical drafts.

## Consequences

The v5 manuscript cannot become a final rules source by implication. Conflicts and omissions become explicit questions rather than a reason to merge versions.
