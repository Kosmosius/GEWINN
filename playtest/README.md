# Playtest workflow

Create a session report from `templates/session-report.md` for every playtest.
Record the exact rules commit and manuscript version before play begins; do not
retrofit findings to a later rules revision.

Use `templates/rules-issue.md` for reproducible defects and ambiguous procedure
interactions. Use `templates/balance-observation.md` for evidence that may be a
balance concern but is not yet a confirmed bug. Add filled reports under
`sessions/` and issue records under `issues/`; this bootstrap creates no
playtest finding.
